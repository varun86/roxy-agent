## 2. 端到端请求链路（当前行为）

1. 前端为当前页面会话生成 session_id，并在每次请求附带 messages 历史。
2. APP/api/app.py 将 session_id 与 messages 透传给 ChatService。
3. ChatService 根据 session_id 从 SessionContextStore 加载上下文。
4. ChatService 把 conversation_history、pinned_skills、compact_summary 传给 HarnessClient。
5. HarnessClient 构建 system instructions：skill_system + context_governance。
6. AsyncAgentLoop 按 system -> history -> current_user 组装消息并调用模型。
7. 回合结束后 ChatService 触发 update_after_turn() 持久化会话上下文。
8. 流式与非流式都走相同治理逻辑（仅事件输出方式不同）。

## 3. 会话上下文数据结构

当前每个 session 对应一个 JSON 文件，核心字段：
1. session_id: 会话标识
2. recent_messages: 最近 user/assistant 消息（过滤 tool 噪声）
3. pinned_skills: 会话内已命中的技能名
4. compact_summary: 历史压缩摘要
5. updated_at: 更新时间

说明：
1. session_id 会做文件名安全归一化。
2. 若文件损坏或不存在，自动回退为空上下文。

## 4. Skills 复用策略（当前版本）

1. 用户消息出现 skill 意图词（如 skill/load/使用/加载）时触发技能命中检测。
2. 如果消息中包含可用技能名，则写入 pinned_skills（去重并限制最大长度）。
3. 下次构建 system prompt 时，在 context_governance 中提示模型优先复用已命中技能。

当前策略特征：
1. 轻量规则匹配，易理解、易调试。
2. 不依赖额外检索组件。
3. 可作为后续语义匹配/检索增强的基础层。

## 5. 压缩策略（当前版本）

1. recent_messages 先按 max_recent_messages 截断。
2. 若截断后字符总量超过 compact_threshold_chars，则把旧半段压缩为摘要行。
3. 摘要累积到 compact_summary，新半段继续保留为 recent_messages。

这是字符阈值近似策略，不是 token 级精准压缩。

## 7. 兼容性与边界

兼容性：
1. 旧请求仅传 message/model 仍可正常运行。
2. 没有 session_id 时按原有无状态逻辑执行。

当前边界：
1. 当前是会话级记忆，不是长期用户记忆。
2. session_id 目前由前端页面生成，刷新页面后会话重置。
3. skill 命中为规则匹配，复杂语义场景可能漏召回。
4. compact 为轻量规则压缩，尚未做 token 精算与质量评估。
