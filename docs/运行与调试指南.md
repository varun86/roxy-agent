# 运行与调试指南（中文）

本文档给你一个最短可行路径：后端跑起来、前端联调、常见问题快速定位。

## 1. 运行前准备

1. Python 3.12+
2. 已安装 uv
3. 已在项目根目录

## 2. 安装依赖

```bash
uv sync
```

依赖定义见：
- [pyproject.toml](../pyproject.toml)

## 3. 配置环境变量

至少要设置：

```bash
export MINIMAX_API_KEY="你的key"
```

可选参数：

```bash
export HARNESS_DEFAULT_MODEL="minimax-m2.7"
export HARNESS_MINIMAX_MODEL="MiniMax-M2.7"
export HARNESS_MINIMAX_BASE_URL="https://api.minimaxi.com/v1"
export HARNESS_SANDBOX_ROOT="/绝对路径/或相对路径"
export HARNESS_SANDBOX_TIMEOUT="60"
export HARNESS_MAX_STEPS="8"
export HARNESS_MAX_OUTPUT_CHARS="12000"
export HARNESS_SUBAGENTS_ENABLED="true"
export HARNESS_SUBAGENTS_MAX_CONCURRENT="3"
export HARNESS_SUBAGENTS_TIMEOUT_SECONDS="900"
```

配置加载逻辑见：
- [harness/config/settings.py](../harness/config/settings.py)

## 5. 启动后端 API

方式一：直接运行模块文件

```bash
uv run python APP/api/app.py
```

默认监听：
- http://0.0.0.0:8000

健康检查：

```bash
curl http://localhost:8000/health
```

查看后端注册模型：

```bash
curl http://localhost:8000/models
```

聊天接口示例：

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"请列出当前目录文件", "model":"minimax-m2.7"}'
```

接口实现见：
- [APP/api/app.py](../APP/api/app.py)

Swagger 调试地址：
- http://localhost:8000/swagger

流式接口示例：

```bash
curl -N -X POST http://localhost:8000/chat/stream \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{"message":"请并行分析三个方向并汇总", "thread_id":"debug-thread-1"}'
```

当前 SSE 可能出现的事件类型：
1. `start`
2. `delta`
3. `task_started`
4. `task_running`
5. `task_completed`
6. `task_failed`
7. `task_timed_out`
8. `done`
9. `error`

## 8. 常见问题与定位

### 问题 1：Environment variable OPENAI_API_KEY is required

原因：未设置对应模型的密钥环境变量（当前默认是 MINIMAX_API_KEY）。

处理：
1. export MINIMAX_API_KEY=...
2. 重新运行命令

### 问题 2：Path is outside sandbox root

原因：访问了 sandbox 根目录以外的路径。

处理：
1. 使用相对路径（相对 HARNESS_SANDBOX_ROOT）
2. 或调整 HARNESS_SANDBOX_ROOT

相关代码：
- [harness/sandbox/runtime.py](../harness/sandbox/runtime.py)

### 问题 3：Unknown tool: xxx

原因：模型调用了未注册工具。

处理：
1. 在 ToolRegistry 注册对应工具
2. 或调整提示词，限制工具使用范围

相关代码：
- [harness/tools/registry.py](../harness/tools/registry.py)
- [harness/tools/executor.py](../harness/tools/executor.py)

### 问题 4：达到 max_steps 仍未结束

现象：返回文本带 “Stopped because max_steps was reached.”

处理：
1. 提高 HARNESS_MAX_STEPS
2. 优化提示词，让模型更快收敛

相关代码：
- [harness/agents/loop.py](../harness/agents/loop.py)

### 问题 5：task 工具报 Nested subagents are disabled

原因：当前版本只允许主 agent 调用 `task`，subagent 内再次委派会被禁止。

处理：
1. 检查是否在 subagent prompt 中错误暴露了 `task`
2. 保持单层委派设计

相关代码：
- [harness/tools/registry.py](../harness/tools/registry.py)
- [harness/client.py](../harness/client.py)

### 问题 6：subagent 超时

现象：
1. SSE 中出现 `task_timed_out`
2. trace 中可能伴随 subagent_errors 增加

处理：
1. 提高 `HARNESS_SUBAGENTS_TIMEOUT_SECONDS`
2. 降低单个子任务复杂度
3. 降低子任务批量数量，让主 agent 分批委派

相关代码：
- [harness/subagents/executor.py](../harness/subagents/executor.py)
- [harness/client.py](../harness/client.py)

## 9. 建议的调试顺序

推荐按下面顺序排查：
1. 先跑 [tests/test_tools.py](../tests/test_tools.py)
2. 再跑 [tests/test_sandbox.py](../tests/test_sandbox.py)
3. 再跑 [tests/test_agent_loop.py](../tests/test_agent_loop.py)
4. 再跑 [tests/test_subagent_limit.py](../tests/test_subagent_limit.py)
5. 再跑 [tests/test_chat_service.py](../tests/test_chat_service.py)
6. 最后跑 APP API（/chat, /chat/stream）接口

这样可以从“局部模块”到“完整链路”逐层定位问题。
